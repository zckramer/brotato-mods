# AI Documentation Repository

This hidden folder contains all AI-oriented documentation for the Brotato modding project.

## Purpose

Centralized reference materials for AI assistants working on Brotato mods, including:

- Game system architecture
- Development workflows
- API references
- Data models
- Best practices

## Structure

### Core Game Systems

- **GAME_SYSTEMS.md** - Complete reference for Brotato's internal systems (RunData, weapons, entities, etc.)
- **WEAPON_ARCHITECTURE.md** - Deep dive into weapon cooldown systems and HUD integration

### Development Processes

- **THIRD_PARTY_ANALYSIS_WORKFLOW.md** - Workflow for analyzing and documenting third-party mods
- **Analytato_DEVELOPMENT.md** - Development guide for Analytato analytics mod

### Data Models

- **DATA_MODEL.md** - Standardized data structure for Analytato stats tracking

### AI Context

- **AI_CONTEXT.md** - Session context and conversation continuity
- **AI_GODOT_REFERENCE.md** - GDScript syntax and Godot engine reference
- **AI_CHECKIN_PROMPT.md** - Prompt for AI session initialization

## Usage

AI assistants should reference these documents when:

- Starting new mod development
- Debugging existing code
- Understanding game systems
- Following established patterns
- Maintaining consistency across sessions

## Maintenance

Keep docs synchronized with actual implementation. Update when:

- New systems are discovered
- Patterns change
- Data models evolve
- Workflows improve
